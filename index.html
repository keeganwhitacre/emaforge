Export Study
Choose an export format. Both options produce a fully self-contained study that runs without a server.
Two-file export (recommended)
Downloads index.html + config.json as a zip. The HTML is lean — it reads config at runtime. Updating study parameters only requires replacing config.json; you don't need to re-export the HTML.
Single-file export
Bakes config inline as a <script> block in the HTML. One file to deploy or send. Use when simplicity of distribution matters more than updateability.
